**[P5-2] apiFetch 반환 객체화 + AbortController** ✅ 2026-04-22 완료 (절충 구현)
- ✅ `apiFetchFull()` 신설 — `{ok, data, error, status, aborted}` 반환
- ✅ `apiFetch()` 는 기존 contract(data|null) 유지하면서 내부적으로 apiFetchFull 재사용 (50+ 호출부 churn 회피)

- ✅ `saveComparison` raw fetch 제거 → apiFetchFull 통합 (TOKEN_EXPIRED 분기는 status===401 로 판단)

1. **보안**: `.git/config` 에 GitHub PAT 평문 저장 상태 — 작업 전 revoke + credential helper 전환 권장
   ```
   git remote set-url origin https://github.com/bji1062/job_change.git
   # GitHub Settings → Developer settings → Personal access tokens 에서 해당 PAT revoke
   ```
2. **미커밋 변경**: `.claude/settings.json`, `server/tools/requirements.txt` 는 오늘 내가 건드린 게 아니라 이전 세션의 변경분 — 커밋 여부는 내일 본인이 판단
3. **systemd 재시작 필요 시점**: 서버 코드 변경(Phase 4 BE 작업들) 후 `sudo systemctl restart jobchoice` 필수. 이미 배포된 변경은 재시작 불필요.
4. **DB 마이그레이션**: 새 마이그레이션 추가 시 `server/seed/migrations/{YYYYMMDD}_{설명}.sql` 네이밍 유지. 적용은 `mysql -u jobapp -pjobapp jobchoice < <파일>`.
