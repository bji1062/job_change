# /research-benefits — 회사 복지 자동 조사

사용자가 `/research-benefits {회사명}` 을 실행하면 아래 단계를 수행합니다.

## 실행 단계

### 1단계: 웹 검색 (WebSearch 3회)

다음 3개 쿼리를 **병렬**로 실행합니다:

```
"{회사명} 복지 혜택 복리후생 2025"
"{회사명} 연봉 복지 잡플래닛 블라인드"
"{회사명} 채용 근무환경 워라밸 재택근무"
```

### 2단계: LLM 파싱 — 7개 카테고리 구조화

검색 결과에서 복지 항목을 추출하여 다음 카테고리로 분류합니다:

| 카테고리 코드 | 한국어 | 포함 항목 예시 |
|--------------|--------|---------------|
| `money` | 금전적 | 식대, 교통비, 복지포인트, 성과급, 경조사비 |
| `health` | 건강 | 건강검진, 체력단련비, 의료비, 단체보험 |
| `housing` | 주거 | 주택자금 대출, 사택, 기숙사 |
| `edu` | 교육 | 교육비, 도서비, 자격증, 컨퍼런스 |
| `family` | 가족 | 자녀학자금, 어린이집, 경조사, 육아휴직 |
| `life` | 생활 | 할인, 동호회, 사내카페, 여행지원 |
| `leave` | 휴가/근무 | 유연근무, 재택, 자율휴가, 리프레시 |

각 항목의 출력 형식:
```json
{
  "key": "meal",
  "name": "식대 지원",
  "val": 180,
  "cat": "money",
  "badge": "est",
  "note": "월 15만원 (추정)",
  "qual": false,
  "qualText": null,
  "sortOrder": 0
}
```

**규칙:**
- `val`: 연간 환산 금액 (만원). 확인 불가 시 업계 평균 추정값 사용
- `badge`: 항상 `"est"` (웹 조사 추정치)
- `qual`: 금액 환산 불가능한 정성적 항목은 `true` (유연근무, 자율휴가 등)
- `qualText`: 정성적 항목의 상세 설명
- `note`: 금액 산출 근거 또는 부가 설명

### 3단계: 사용자 확인

파싱 결과를 테이블 형태로 사용자에게 보여줍니다:

```
## {회사명} 복지 조사 결과 (WebSearch 기반, badge=est)

| # | 카테고리 | 항목 | 연간 (만원) | 비고 |
|---|---------|------|-----------|------|
| 1 | 금전적  | 식대 지원 | 180 | 월 15만원 추정 |
| 2 | 건강    | 건강검진 | 100 | 본인+가족 |
...

합계: 약 {총액}만원/연

이 결과를 DB에 저장하시겠습니까?
```

AskUserQuestion 도구로 확인을 요청합니다.

### 4단계: DB 저장

사용자가 확인하면:

1. 회사가 DB에 있는지 확인 (`GET /api/v1/companies/search?q={회사명}`)
2. 없으면 회사 등록이 필요하다고 안내
3. 있으면 `PUT /api/v1/companies/{company_id}/benefits` 로 저장
   - 기존 `est` badge 항목만 덮어씌움 (`official` 보존)

### 4-1단계 (옵션): 채용 홈페이지 직접 접근

DB의 `careers_benefit_url` 컬럼에 URL이 있고, Puppeteer MCP가 설정되어 있으면:
1. Puppeteer로 채용 홈페이지 복지 페이지 접근
2. 페이지 텍스트 추출 → LLM 파싱
3. WebSearch 결과와 병합 (교차 검증, 중복 제거)
4. Puppeteer 출처 항목은 `badge: "official"` 로 표시

현재 환경에서는 Puppeteer MCP 미설정이므로 WebSearch만 사용합니다.

## 주의사항

- 금액은 추정치입니다. 사용자에게 `badge=est` 임을 명확히 안내합니다.
- 검색 결과가 부족한 경우, 기업유형별 `benefit_presets` 를 폴백으로 제안합니다.
- 소규모/비상장 회사는 정보가 부족할 수 있음을 안내합니다.
